## Product Image Batch Pipeline (Next.js-ready)

This project includes a free batch script for product image preparation:

- Background removal into `bg_removed` (uses `rembg` if available)
- `cleaned` stage for manual touch-up override
- Web-ready exports into `web_ready` as square `webp`
- Direct copy to `public/products/generated` for Next.js
- Manifest JSON output at `web_ready/manifest.nextjs.json`

### Run (PowerShell)

```powershell
cd "D:\Success_Suphancasting\suphancasting"
powershell -ExecutionPolicy Bypass -File ".\scripts\run-product-image-pipeline.ps1" -InstallDeps
```

### Optional custom run

```powershell
powershell -ExecutionPolicy Bypass -File ".\scripts\run-product-image-pipeline.ps1" `
  -SourceRoot "D:\Success_Suphancasting\Product_image" `
  -NextJsPublicDir "D:\Success_Suphancasting\suphancasting\public\products\generated" `
  -Size 1200 `
  -Quality 85 `
  -Background transparent
```

### Recommended standards

- Format: `webp`
- Size: `1200x1200`
- Quality: `80-85`
- Naming: lowercase slug (auto-generated by script)
- Next.js source path: `/products/generated/<file>.webp`

## Two-Folder Auto Sync (recommended)

If you want exactly two folders:

1. `D:\Success_Suphancasting\Product_image\original` (drop any source files here)
2. `D:\Success_Suphancasting\Product_image\web` (auto generated web-ready files)

Use this script to auto-sync and improve images:

```powershell
cd "D:\Success_Suphancasting\suphancasting"
powershell -ExecutionPolicy Bypass -File ".\scripts\run-two-folder-sync.ps1" -InstallDeps -Watch
```

What it does:

- Detects new/updated files in `original`
- Removes background (if `rembg` is installed)
- Resizes to square web standard (`1200x1200`)
- Exports `.webp` files into `web`
- Copies output to `public/products/generated` for Next.js
- Generates manifest at `Product_image\web\manifest.nextjs.json`
